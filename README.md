# Учёт личных финансов (PersonalFinance)

Учебный проект по профессиональному модулю **ПМ.02 «Осуществление интеграции
программных модулей»** (производственная практика ПП.02.01). Десктопное
приложение на **C# / .NET 9 / Windows Forms** для учёта личных доходов и
расходов, ведения счетов и контроля месячных бюджетов.

Выполнил: **Маслов Данила Викторович**, группа ИСс-31, специальность
09.02.07 «Информационные системы и программирование».

## Архитектура

Приложение построено по схеме **MVC** (Model–View–Controller) с разделением на слои:

| Слой | Каталог | Назначение |
|------|---------|------------|
| Модель | `Models/` | Сущности предметной области и перечисления |
| Сервисы | `Services/` | Хранение данных, валидация, отчёты, логирование, отмена операций |
| Контроллеры | `Controllers/` | Бизнес-логика, связь моделей и форм |
| Представление | `Forms/` | Пользовательский интерфейс (WinForms) |

## Применённые паттерны проектирования

| Паттерн | Реализация | Назначение |
|---------|------------|------------|
| Наблюдатель | события `DataChanged` / `StatusChanged` | обновление UI при изменении данных |
| Фабричный метод | `ReportFactory` → `IReport` (PDF/Excel/CSV) | создание генератора отчёта нужного формата |
| Стратегия | `ITransactionFilter` + `TransactionFilterContext` | гибкая фильтрация списка операций |
| Снимок (Memento) | `FinanceMemento`, `FinanceState`, `UndoManager` | отмена и повтор операций (Ctrl+Z / Ctrl+Y) |

## Запуск

1. Открыть `PersonalFinance.sln` в Visual Studio 2022 (рабочая нагрузка
   «.NET Desktop Development») или выполнить `dotnet run` из каталога проекта.
2. При первом запуске автоматически создаются демонстрационные данные.
3. Окно входа — демо-доступ:
   - `admin` / `admin`
   - `user` / `123456`

## Возможности

- Операции: добавление, редактирование, удаление, поиск и фильтрация по
  категории и типу (доход/расход).
- Счета (кошельки): ведение нескольких счетов с автоматическим пересчётом баланса.
- Бюджеты: месячные лимиты расходов по категориям с расчётом фактических затрат
  и процента использования.
- Отчёты в форматах **PDF, Excel и CSV** (паттерн «Фабрика»).
- Статистика расходов по категориям (отрисовка средствами GDI+).
- Автосохранение каждые 30 секунд и сохранение при закрытии.
- Отмена и повтор операций (Ctrl+Z / Ctrl+Y).
